## 완전범죄

### 풀이

범위가 작다고 생각해서 그냥 브루트포스로 풀었다가 시간초과가 나서 **DP**구나 싶었다.  
그리디로 풀기에는 데이터를 봤을 때 최적해가 나오지 않는다고 판단했다.  
2차원 배열을 사용해서 n번째 물건에서 B의 흔적(0~m-1)까지의 수치에 해당하는 A의 흔적의 최솟값을 구해 최적해를 도출하도록 코드를 작성했다.  
한 번에 풀이 방식이 생각나지는 않았고 지피티에게 문제를 풀이할 힌트를 달라고 했다. DP 문제는 항상 보면 DP인 건 아는데 풀이가 어렵다. 더 연습이 필요하다고 생각한다.

### 코드

```java
class Solution {
    public int solution(int[][] info, int n, int m) {
        // [훔치는 물건 index][B의 흔적값]
        int[][] dp = new int[info.length + 1][m];
        int INF = 100_000;

        for (int i = 0; i <= info.length; i++) {
            for (int j = 0; j < m; j++) {
                dp[i][j] = INF;
            }
        }

        dp[0][0] = 0;

        for (int i = 0; i < info.length; i++) { // 0번째 물건부터 info.length-1번째 물건까지 순회
            for (int j = 0; j < m; j++) { // B의 흔적
                if (dp[i][j] == INF) continue; // 가능하지 않은 경우

                //i번째 물건을 A가 훔치는 경우
                dp[i + 1][j] = Math.min(dp[i + 1][j], dp[i][j] + info[i][0]);

                if (j + info[i][1] < m) { // i번째 물건을 B가 훔치는 경우
                    dp[i + 1][j + info[i][1]] = Math.min(dp[i + 1][j + info[i][1]], dp[i][j]);
                }
            }
        }

        // 모든 물건을 훔쳤을 때 가장 작은 A의 흔적 값
        int min = INF;
        for (int i = 0; i < m; i++) {
            min = Math.min(min, dp[info.length][i]);
        }

        // 최대치를 넘겼다면 모든 물건을 훔칠 수 없다
        return min >= n ? -1 : min;
    }
}
```
